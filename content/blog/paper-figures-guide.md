用 AI 生成 Python matplotlib 程式碼，在 Google Colab 執行，輸出 SVG 向量圖。不需要安裝任何軟體、不需要會寫程式、不需要花錢買繪圖工具。這個流程產出的圖表直接符合 SCI 期刊投稿規範。

## 為什麼不能用 Excel 截圖

Excel 圖表貼進 Word 再截圖投稿——這是最常見的做法，也是 reviewer 最容易抓的問題：

- **解析度不足**：截圖是點陣圖（72-96 DPI），期刊要求 300 DPI 以上
- **放大會糊**：排版時圖表會被縮放，點陣圖一放大就模糊
- **格式不規範**：Excel 預設的顏色、字體、圖例位置都不符合學術慣例
- **無法編輯**：reviewer 要求你改顏色或加標注，你得從頭做

向量圖（SVG、PDF、EPS）沒有這些問題。它無限放大不失真、檔案小、隨時可修改。而 Python matplotlib 就是產出向量圖最直接的工具。

## AI 圖表工作流

整個流程四步：

### 第一步：描述你要什麼圖

把以下資訊告訴 AI：

```
我需要一張學術圖表：
- 圖表類型：[分組長條圖 / 折線圖 / 散佈圖 / 箱形圖 / 熱力圖]
- 數據：[直接貼數據，或描述數據結構]
- X 軸：[變數名稱與單位]
- Y 軸：[變數名稱與單位]
- 分組：[如果有的話]
- 顯著性標記：[需要標 * 或 p-value 嗎]
- 輸出格式：SVG
- 特殊要求：[色盲友善 / 灰階列印友善 / 期刊指定尺寸]

請生成 Python matplotlib 程式碼，
使用學術風格（白底、serif 字體、無多餘裝飾），
圖表尺寸設為期刊單欄寬度（3.5 inch）或雙欄寬度（7 inch）。
```

### 第二步：在 Google Colab 執行

把 AI 生成的 Python 程式碼整段複製到 [Google Colab](https://colab.research.google.com/)，點 Run。如果程式碼報錯，把錯誤訊息貼回給 AI，它會幫你修。通常一兩輪就能跑出正確的圖。

### 第三步：檢查並微調

圖出來之後，檢查以下幾點：

- 標籤文字會不會太小？（最小 8pt，建議 10pt）
- 圖例位置會不會擋住數據？
- 顏色在灰階列印下能不能區分？
- 顯著性標記的位置對不對？

不滿意的地方直接告訴 AI：「把字體放大到 12pt」、「把圖例移到右上角」、「把顏色換成色盲友善色板」。它會修改程式碼，你再跑一次。

### 第四步：下載正確格式

```python
# SVG（向量圖，推薦）
plt.savefig('figure1.svg', bbox_inches='tight', dpi=300)

# PDF（向量圖，部分期刊偏好）
plt.savefig('figure1.pdf', bbox_inches='tight', dpi=300)

# TIFF（某些醫學期刊要求）
plt.savefig('figure1.tiff', bbox_inches='tight', dpi=300,
            pil_kwargs={'compression': 'tiff_lzw'})

# PNG（通用點陣圖，至少 300 DPI）
plt.savefig('figure1.png', bbox_inches='tight', dpi=300)
```

## 圖表類型選擇指南

不同的研究問題對應不同的圖表類型。選錯圖表類型比圖做得醜更嚴重——它會誤導讀者。

| 你想呈現的 | 建議圖表 | 常見錯誤 |
|-----------|---------|---------|
| 組間比較 | 分組長條圖（附誤差線） | 用圓餅圖比較組間差異 |
| 時間趨勢 | 折線圖 | 用長條圖畫時間序列 |
| 兩變數關係 | 散佈圖 | 只報相關係數不畫圖 |
| 數據分佈 | 箱形圖或小提琴圖 | 只用平均值長條圖（隱藏分佈資訊） |
| 流程或架構 | 流程圖（用 draw.io 或 AI SVG） | 用 PowerPoint 手畫 |
| 多組多變數 | 熱力圖 | 用一堆長條圖並排 |

一個常被忽略的原則：**箱形圖幾乎永遠優於長條圖**。長條圖只顯示平均值和誤差線，箱形圖還能顯示中位數、四分位距、離群值。Reviewer 越來越偏好看到數據分佈，而不是只看平均值。

## 色彩語義一致性

整篇論文中，同一個變數或組別必須使用同一個顏色。如果圖 1 中「實驗組」是藍色，圖 3 中「實驗組」不能變成紅色。

在程式碼中定義一個全域色板：

```python
COLORS = {
    'experimental': '#2171B5',  # 藍
    'control': '#CB181D',       # 紅
    'baseline': '#6A6A6A',      # 灰
}
```

每張圖都引用這個色板，確保一致性。

推薦色盲友善色板：

- **Blue-Orange**：`#2171B5` + `#E6550D`（最安全的兩色組合）
- **ColorBrewer Set2**：適合 3-8 組的分類資料
- **Viridis**：適合連續色階（熱力圖）

## 多面板組合圖

很多期刊限制圖表數量（例如最多 6 張），所以你需要把相關的子圖合成一張。AI 可以直接生成多面板圖：

```
請生成一張 2×2 多面板圖：
(a) 左上：各組平均值比較（長條圖）
(b) 右上：時間趨勢（折線圖）
(c) 左下：變數相關性（散佈圖）
(d) 右下：數據分佈（箱形圖）

每個子圖左上角標注 (a)(b)(c)(d)，
共用一個 X 軸標籤，整體尺寸 7×6 inch。
```

AI 會用 `plt.subplot()` 或 `fig.add_subplot()` 幫你排版。子圖標籤 (a)(b)(c)(d) 是學術規範，不能省。

## 投稿前圖表檢查清單

投稿前逐項確認：

- [ ] **向量格式**：SVG、PDF 或 EPS（不是 PNG 截圖）
- [ ] **解析度**：如果用點陣圖，至少 300 DPI（線條圖 600 DPI）
- [ ] **完整標籤**：X 軸、Y 軸都有名稱和單位
- [ ] **圖例清楚**：不會擋住數據，文字可讀
- [ ] **字體大小**：最小 8pt，縮放後仍可辨識
- [ ] **色盲友善**：不要只靠顏色區分，搭配不同線型或標記
- [ ] **顯著性標記**：`*` p < 0.05, `**` p < 0.01, `***` p < 0.001, `ns` 不顯著
- [ ] **色彩一致**：全文同一變數同一顏色
- [ ] **白底無框**：移除灰色背景和外框（matplotlib 預設有灰底）
- [ ] **符合期刊要求**：尺寸、格式、色彩模式（RGB / CMYK）

## FAQ

### 完全不會寫程式怎麼辦？

不需要會。整個流程是：告訴 AI 你要什麼圖 → AI 給你 Python 程式碼 → 複製到 Google Colab → 點 Run → 下載圖片。你不需要理解程式碼的每一行，只需要能描述你要什麼。如果程式碼報錯，把錯誤訊息貼給 AI，它會修好。

### 期刊要求 TIFF 格式怎麼辦？

在 `plt.savefig()` 裡指定格式就好。AI 生成的程式碼通常預設輸出 PNG，你只要把副檔名改成 `.tiff` 並加上壓縮參數。上面的程式碼範例已經包含 TIFF 輸出。

### 已經做好 Excel 圖了，不想重做？

把 Excel 圖截圖給 AI，告訴它：「請根據這張圖重新生成符合學術規範的 Python matplotlib 程式碼，保持相同的數據和圖表類型。」AI 會辨識截圖中的數據並重新繪製。你只需要校對數字是否正確。

### 流程圖和架構圖也能用 Python 畫嗎？

可以但不推薦。流程圖用 [draw.io](https://app.diagrams.net/)（免費、輸出 SVG）或請 AI 生成 Mermaid 語法更方便。Python matplotlib 擅長的是數據圖表，不是示意圖。
