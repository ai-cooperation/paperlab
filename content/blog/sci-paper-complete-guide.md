投稿是包裝工程，不是寫完 PDF 上傳就好。選錯期刊、漏附件、Cover Letter 寫成自我介紹——這些都會讓你在 editor 手上就被退稿（desk reject），連 reviewer 都見不到。

這篇文章給你三個工具：選期刊的五維框架、投稿前的 10 項 checklist、Cover Letter 的五段結構。照著做，你可以把投稿準備時間從兩週壓縮到三天。

## 你可能正在經歷這些

> 「印了期刊格式要求 10 幾頁 A4，每一行都在告訴你還缺什麼。」

> 「TSSCI 引用好難，每個期刊都不一樣，光改格式就花了三天。」

格式焦慮是真實的，但格式不是投稿最難的部分。最難的是「策略」——投哪本、怎麼包裝、怎麼讓 editor 三十秒內決定不退你的稿。格式問題有工具可以解決，策略問題需要框架。

## 一、選期刊：五維決策框架

不要「寫完再找期刊」，也不要「只投最好的期刊」。選期刊是一個多維度的決策，以下五個維度按優先順序排列：

### 1. Scope Match（最重要）

期刊的 Aims & Scope 必須明確涵蓋你的研究主題。這不是「大概相關」就好——如果你的論文需要 editor 想一下「這算不算我們的範圍」，desk reject 的機率就很高。

**實際做法**：讀目標期刊最近兩年發表的論文標題，看有沒有跟你的研究在同一個子領域的。有就代表 scope match，沒有就別勉強。

### 2. Impact（影響力）

看 Impact Factor 和 CiteScore，但不要只看數字。同一個領域內，IF 3.0 的期刊可能比另一個領域 IF 5.0 的期刊更難投。看 JCR 的分區（Q1-Q4）比看絕對數字更有意義。

對碩士生和博一生的務實建議：第一篇論文以 Q2-Q3 為目標。Q1 的競爭激烈，審稿週期長，退稿後你可能沒有時間重投。Q2-Q3 的接受率更高，審稿更快，拿到第一篇發表後再往上衝。

### 3. Review Speed（審稿速度）

在 Scopus 或 Web of Science 上查期刊的「average review time」。有些期刊從投稿到首次決定要 6-8 個月，有些只要 4-6 週。如果你有畢業時限，審稿速度是硬限制。

也可以到 [SciRev](https://scirev.org) 看其他人回報的實際審稿時間，比期刊官方數據更準。

### 4. Acceptance Rate（接受率）

大多數期刊不公開接受率，但你可以從幾個指標推估：每年發表的論文數量（量大通常接受率較高）、期刊有沒有 Special Issue（SI 通常接受率更高）、reviewer 的評語風格（建設性 vs. 找碴型，看 Publons 上的公開評審）。

### 5. Open Access Policy

如果你的研究經費有限，注意 OA 費用。Gold OA 的 APC（Article Processing Charge）從 500 到 5000 美元都有。很多期刊提供 Green OA（可以在機構 repository 放 preprint），不用付費。有些期刊對開發中國家作者提供 APC 減免，投稿前先查。

## 二、投稿 10 項 Checklist

分成「稿件五項」和「附件五項」，缺任何一項都可能被退回要求補件。

### 稿件五項

1. **Title Page**：標題、所有作者姓名與機構、通訊作者 email、ORCID。注意作者順序——投稿後改作者順序非常麻煩，事前跟共同作者確認清楚。
2. **Abstract**：依期刊要求的字數限制（通常 150-300 字）。結構式摘要（Background-Method-Result-Conclusion）或非結構式，看期刊規定。
3. **Manuscript Body**：IMRaD 結構（Introduction, Methods, Results, and Discussion）。確認 section 標題跟期刊範本一致。
4. **References**：格式必須跟期刊要求完全一致。用 Zotero 或 Mendeley 管理，選對 citation style。如果你用 Markdown + Quarto 寫論文，換期刊只需要改 YAML header 裡的 `csl` 欄位。
5. **Figures & Tables**：解析度至少 300 DPI。確認圖表編號在文中有對應引用。有些期刊要求圖表另外上傳，不嵌在稿件裡。

### 附件五項

6. **Cover Letter**：下一節詳述。
7. **Highlights**：3-5 條，每條不超過 85 字元。用「動詞開頭」的句式（Proposed... / Demonstrated... / Revealed...）。
8. **Graphical Abstract**：不是每個期刊都要求，但有的話會增加曝光度。尺寸通常有嚴格規定（例如 Elsevier 要求 531 x 1328 pixels）。
9. **Supplementary Materials**：大表格、額外實驗、程式碼連結。確認 supplementary 裡的圖表有自己的編號系統（Table S1, Figure S1）。
10. **Author Agreement / Conflict of Interest**：有些期刊要求所有作者簽名的同意書和利益衝突聲明。提前準備，不要到上傳系統才發現還要找共同作者簽名。

## 三、Cover Letter 五段結構

Cover Letter 是 editor 看到的第一份文件。一封好的 Cover Letter 能讓 editor 在 30 秒內理解你的論文為什麼值得送審。

### 第一段：開場定位（2-3 句）

說明你投的是什麼類型的稿件（Original Research / Review / Short Communication），論文標題，以及你想投的期刊名稱。

> Dear Editor, We would like to submit our original research article entitled "..." for consideration in [Journal Name].

### 第二段：研究問題與重要性（3-4 句）

用一句話說明你要解決的問題，再用一兩句話說明為什麼這個問題重要。不要從「近年來 AI 發展迅速」這種廢話開頭——editor 不需要你介紹他自己的領域。直接講問題。

### 第三段：你的貢獻（3-4 句）

具體說明你做了什麼、發現了什麼。用數字：「我們的方法在 X 數據集上達到 Y% 的準確率，比 SOTA 高 Z 個百分點」。這是 Cover Letter 最重要的段落。

### 第四段：為什麼選這本期刊（1-2 句）

說明你的研究跟這本期刊的 scope 如何吻合。如果可以引用該期刊近期發表的相關論文就更好——這表示你真的讀過這本期刊，不是亂槍打鳥。

### 第五段：標準結尾（2-3 句）

確認論文沒有在其他地方投稿或發表、所有作者同意投稿、沒有利益衝突。署名通訊作者。

## 用 Markdown 寫論文，換期刊改一行

如果你用 Word 寫論文，每次換期刊都是一場災難：標題字體、行距、引用格式、圖表位置全部要重排。

如果你用 Markdown + [Quarto](https://quarto.org) 寫論文，換期刊只需要改 YAML header：

```yaml
# 投 Elsevier
format:
  elsevier-pdf:
    journal:
      name: "Expert Systems with Applications"
csl: elsevier-harvard.csl

# 改投 Springer
format:
  springer-pdf:
    journal:
      name: "Neural Computing and Applications"
csl: springer-basic-author-date.csl
```

一行 `csl` 換引用格式，一行 `format` 換版面。重新 render 就好，不用手動調三天。初期學習 Markdown 需要幾小時，但從第二次投稿開始你就會回本。

## FAQ

### Cover Letter 到底重不重要？

非常重要。Editor 每天收到幾十篇投稿，Cover Letter 是他判斷「要不要花時間看 abstract」的第一道關卡。一封空洞的 Cover Letter（「Please find attached our manuscript...」然後就沒了）等於在告訴 editor 你不在乎這本期刊。

### 被 A 期刊退稿，投 B 期刊要全部重排版嗎？

如果你用 Word：很可能要。引用格式、section 標題、圖表規範每本期刊都不同。如果你用 Markdown + Quarto：改 YAML header 裡的期刊模板和 CSL 檔，重新 render，十分鐘搞定。這就是為什麼投資學 Markdown 寫作流程是值得的。
