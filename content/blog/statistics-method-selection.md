統計恐懼的根源不是數學能力不足，而是「方法選擇」問題。你不需要懂背後的數學推導，你需要學會一件事：精確描述你的數據長什麼樣，讓 AI 幫你選方法、寫程式碼、解讀結果。

這篇文章給你一張決策表、一套 AI 提問模板、一個三重驗證流程。照著做，統計分析從「恐懼」變成「流程」。

## 你可能正在經歷這些

> 「一直很害怕醫學論文⋯⋯對統計有深深的恐懼。」

> 「統計快掛了⋯⋯文組出身，統計初學者。」

這兩句話的共同點：把統計當成一門需要「精通」的學問。但對論文寫作來說，統計是工具，不是學科。你不需要推導公式，你需要知道什麼情境用什麼工具。就像你不需要懂引擎原理才能開車。

## 第一步：描述你的數據，不是選方法

大多數人的錯誤流程是：先 Google「我該用什麼統計方法」，然後被一堆術語淹沒。

正確流程是反過來的——先搞清楚你的數據長什麼樣，方法自然浮現。你需要回答四個問題：

1. **依變項（你要測量的結果）是什麼類型？** 連續變數（身高、分數、血壓）還是類別變數（通過/不通過、疾病分類）？
2. **自變項（你要比較的分組或預測因子）有幾個？** 一個還是多個？每個是連續的還是類別的？
3. **樣本設計是什麼？** 獨立樣本（不同的人）還是相依樣本（同一批人測兩次）？
4. **樣本數多大？** 每組多少人？

這四個問題的答案決定了你 90% 的方法選擇。

## 第二步：用決策表直接對應

把四個問題的答案對照這張表：

| 情境 | 建議方法 | 前提條件 |
|------|---------|---------|
| 比較兩組平均值（獨立樣本） | 獨立樣本 t-test | 常態分佈、變異數齊性 |
| 比較兩組平均值（同一批人前後測） | 配對 t-test | 差值常態分佈 |
| 比較三組以上平均值 | 單因子 ANOVA | 常態分佈、變異數齊性 |
| 兩個類別變數的關聯 | 卡方檢定 | 每格期望值 ≥ 5 |
| 一個連續依變項 + 多個預測因子 | 多元迴歸 | 殘差常態、無多重共線性 |
| 依變項是二元（是/否） | 邏輯斯迴歸 | 樣本數 ≥ 事件數 × 10 |
| 樣本太小或違反常態假設 | 非參數檢定（Mann-Whitney, Kruskal-Wallis） | 無嚴格前提 |

你不需要背這張表。把它存起來，每次分析前對照一次就好。

## 第三步：讓 AI 幫你選方法並寫程式碼

把你的四個答案直接丟給 AI，用這個模板：

```
我的研究設計如下：
- 依變項：[描述，例如「收縮壓，連續變數」]
- 自變項：[描述，例如「兩組：實驗組 vs 對照組」]
- 樣本設計：[獨立樣本 / 配對樣本]
- 樣本數：[每組人數]
- 數據格式：[CSV / Excel]

請建議適合的統計方法，說明為什麼選這個方法，
並提供 Python 程式碼（使用 scipy 和 statsmodels），
包含：前提檢驗、主要分析、效果量計算、結果解讀。
```

AI 會回傳完整的分析腳本。你不需要自己寫程式碼——把 AI 生成的 Python code 複製到 Google Colab，上傳你的 CSV，按 Run 就能跑。

關鍵在於你描述數據的精確度。描述越模糊，AI 的建議越不可靠；描述越精確，AI 的建議越接近統計教授的判斷。

## 第四步：三 AI 交叉驗證

統計方法的選擇牽涉到判斷，不同 AI 可能給出不同建議。用三個 AI 交叉確認：

1. **Claude**：擅長邏輯推理，會清楚解釋「為什麼選這個方法而不是那個」
2. **ChatGPT**：解釋清晰易懂，適合確認你對結果的理解是否正確
3. **Gemini**：擅長事實查核，可以檢查前提條件是否真的滿足

三個 AI 都建議同一個方法→放心使用。如果有分歧→把分歧點拿去問指導教授，這時候你問的是一個具體問題（「t-test 和 Mann-Whitney 在我這個情境下哪個更適合」），而不是一個模糊的「我該用什麼方法」。

## p-value 到底怎麼解讀

既然講到統計，不能不提 p-value。一句話：**p < 0.05 不代表你的假設是對的，它只代表「如果虛無假設為真，觀察到這種結果的機率很低」**。

實務上你需要報告三件事：

1. **p-value**：顯著性（但別只看這個）
2. **效果量**（Cohen's d、η²、R² 等）：差異有多大
3. **信賴區間**：估計值的不確定性範圍

很多 reviewer 現在會要求報告效果量，只報 p-value 不夠。在你的 AI 提問模板裡已經包含了效果量計算，所以程式碼會自動算好。

## 常見方法比較速查

怕搞混 t-test、ANOVA 和迴歸？它們的關係其實很簡單：

- **t-test**：比較兩組的平均值差異
- **ANOVA**：比較三組以上的平均值差異（t-test 的延伸）
- **迴歸**：看一個或多個變數能不能預測另一個變數

更直白地說：t-test 和 ANOVA 回答「有沒有差」，迴歸回答「能不能預測」。如果你的研究問題是比較組間差異，用前兩個；如果是探討變數之間的關係，用迴歸。

## FAQ

### AI 建議的統計方法可靠嗎？

取決於你描述數據的精確度。如果你只說「我想比較兩組」，AI 可能給出不完整的建議。如果你把依變項類型、樣本設計、樣本數、數據分佈都說清楚，AI 的建議跟統計教科書的決策樹幾乎一致。搭配三重驗證，可靠度更高。

### 完全不會寫程式碼怎麼辦？

不需要會。把 AI 生成的 Python code 整段複製到 [Google Colab](https://colab.research.google.com/)，上傳你的數據檔案，點 Run。Colab 是免費的雲端 Python 環境，不用安裝任何東西。如果程式碼報錯，把錯誤訊息貼回給 AI，它會幫你修。

### 指導教授堅持用 SPSS 怎麼辦？

兩邊都做。用 AI 生成 Python 程式碼跑一次確認結果，再用 SPSS 手動操作一次給教授看。兩邊的數字應該一致——如果不一致，代表某一邊的設定有問題，這反而幫你抓到了錯誤。
